# 文献レビュー

論文メモ、既存研究比較、研究ポジショニング整理に使う。

## ワークフロー
1. 問題設定と dataset / task 前提を正確に特定する。
2. 著者が主張する contribution を抜き出す。
3. method の新規性と engineering detail を分ける。
4. 主張成立に必要な前提を列挙する。
5. 主張を支える evidence を列挙する。
6. 欠けている baseline、failure mode、IntentFlow への transfer risk を整理する。

## 出力フォーマット
- Problem
- Contribution
- Method
- Evidence
- Limitations
- Relevance to IntentFlow
- What would falsify the main claim
