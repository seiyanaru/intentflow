# 2026-03-03 研究メモ（2026-03-04 更新）

## 今回の変更概要
今回の更新では、Neuro-Gated OTTA を **Dual Gating（Neuro + Statistical）** として実装し、
テスト時適応（OTTA）の安全性を高めるためのゲート条件を強化しました。

## 1. 実装方針の変更点
- 旧: `pmax` と `SAL` を主条件に適応判定
- 新: **Tri-Lock 判定**に変更
  - Confidence Gate: `pmax > pmax_th`
  - Physiological Gate: Neuro-Score の Z スコアに応じた閾値引き上げ（Conservative）
  - Statistical Gate: `Energy Score <= energy_th`

## 2. 主要コード変更
### `intentflow/offline/models/pmax_sal_otta.py`
- `energy_threshold`, `energy_quantile`, `energy_temperature`, `neuro_beta`, `strict_tri_lock` を導入
- `compute_energy()` を追加し、`-T*logsumexp(logits/T)` を算出
- `compute_source_prototypes()` 内で Source 由来の Energy 閾値を校正
- `compute_sal()` の fail-safe を修正（プロトタイプ未計算時は `SAL=0` で適応ブロック）
- `compute_gate()` を Tri-Lock 対応に変更
- `energy_score` と `skipped_energy` を統計出力へ追加
- 特徴フックを「分類器入力特徴」を優先して取得するよう修正

### `intentflow/offline/models/tcformer_otta.py`
- OTTA 初期化時に新パラメータを `PmaxSAL_OTTA` へ伝播
- OTTA stats 保存に `energy_score` を追加
- **重要バグ修正**: `set_train_dataloader()` を `TCFormerOTTA` に追加
  - これにより test 前の `compute_source_prototypes()` が確実に実行される

### `intentflow/offline/configs/tcformer_otta/tcformer_otta.yaml`
- 以下を `model_kwargs` に追加
  - `neuro_beta: 0.1`
  - `energy_threshold: null`
  - `energy_quantile: 0.95`
  - `energy_temperature: 1.0`
  - `strict_tri_lock: true`
