# 2026-01-29 Research Progress: Neuro-Gated OTTA Phase 7

## 1. 検証の目的 (What & Why)
**検証テーマ**: **Online Z-score Normalization (オンラインZスコア正規化)**

- **背景**: 
    - 従来の「Neuro-Gating」は、被験者ごとの脳波エネルギー（Neuro-Score）の絶対値にばらつきがあり、固定パラメータ（$\alpha=0.2$ や $50.0$）では制御不能でした。
- **解決策**:
    - **「絶対値」ではなく「偏差値（Zスコア）」を使う** ことで、被験者ごとの個体差を自動的に吸収します。
    - これにより、パラメータ調整（キャリブレーション）なしで、誰にでも安定して動作する適応システムを目指します。

### パラメータ制御式
閾値の動的調整は以下の数式で行われます。

$$
\theta_{dynamic} = \theta_{base} - \beta \cdot \tanh\left(\frac{S - \mu_t}{\sigma_t}\right)
$$

- $S$: 現在のサンプルのNeuro-Score（運動野の活性度）
- $\mu_t, \sigma_t$: 過去のサンプルから更新された移動平均と標準偏差
- $\beta$: 感度パラメータ（Phase 7では $0.1$ に設定）
- **動作**: 
    - 運動想起が強い（$Z > 0$）→ 閾値が下がる → **適応しやすくなる**
    - ノイズが多い（$Z < 0$）→ 閾値が上がる → **適応をブロックする**

## 3. 実装ミスの原因と対策 (Troubleshooting)
今回の検証で発生した問題と、その技術的背景です。

| 問題 (What) | 原因 (Why) | 対策 (Fix) |
|---|---|---|
| **適応率が 17% しかなかった** | `Warm-up=20` に設定していたが、テストデータが少なすぎて（全58サンプル）、最初の48サンプル（Batch 1）が全てウォームアップで捨てられ、残りの10サンプルしか適応判定されなかった。 | **Warm-up期間を「1回」に短縮**し、データが少なくても即座に適応を開始するように修正。 |
| **SALが常に 1.0 (最大値) だった** | コード修正時、学習データから「プロトタイプ（基準点）」を計算する処理を呼び出し忘れていたため、類似度が計算できず、安全策として「1.0」を返すデフォルト動作になっていた。 | **`set_train_dataloader` メソッドを明示的に呼び出す** 処理を追加し、テスト前にプロトタイプを確実に計算させる。 |
