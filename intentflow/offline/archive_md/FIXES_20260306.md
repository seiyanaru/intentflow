# Critical Fixes Applied on 2026-03-06

## Summary
Fixed two critical issues that were preventing proper baseline reproduction:
1. **EarlyStopping causing premature training termination**
2. **Validation leakage (val_dataloader returning test data)**

These issues resulted in:
- Training stopping at ~200-400 epochs instead of 1000 epochs
- Model selection based on test set performance (data leakage)
- Baseline accuracy of 80.98% instead of expected 84.79%

---

## Changes Made

### 1. EarlyStopping Removal

**File: `train_pipeline.py`**

**Before:**
```python
from pytorch_lightning.callbacks import EarlyStopping

early_stopping = EarlyStopping(
    monitor="val_loss",
    mode="min",
    patience=50,
    verbose=True
)

callbacks=[metrics_callback, early_stopping]
```

**After:**
```python
# from pytorch_lightning.callbacks import EarlyStopping  # DISABLED

# EarlyStopping DISABLED: It was causing data leakage because val_dataloader()
# returns test_dataloader() in base.py, meaning the model was stopping based
# on test set performance. This violated paper reproduction protocol.

callbacks=[metrics_callback]  # early_stopping removed
```

**Impact:** Models will now train for full 1000 epochs as specified in config.

---

### 2. Validation Leakage Fix

**File: `base.py`**

**Before:**
```python
def val_dataloader(self) -> DataLoader:
    return self.test_dataloader()
```

**After:**
```python
def val_dataloader(self) -> DataLoader:
    # IMPORTANT: Validation should NOT be the same as test (data leakage)
    if hasattr(self, 'val_dataset') and self.val_dataset is not None:
        return DataLoader(self.val_dataset, ...)
    else:
        # Fallback with warning
        print("WARNING: val_dataset not found. Returning test_dataloader (DATA LEAKAGE!)")
        return self.test_dataloader()
```

**File: `bcic4_2a.py`**

**Changes:**
- Added `val_dataset = None` attribute to BCICIV2a class
- Modified `setup()` to split session_T into train (80%) and validation (20%)
- Session_E is used as test set (when eval labels are available)
- Updated z_scale to use 3-way split: `_z_scale_tvt(X, X_val, X_test)`

**Data Split (when session_E exists):**
- Training: 80% of session_T (~230 trials)
- Validation: 20% of session_T (~58 trials)
- Test: 100% of session_E (288 trials)

**Impact:** Proper train/val/test separation, no data leakage.
